## Bem-vindo! 👋

Obrigado por conferir este desafio de front-end.

[Frontend Mentor](https://www.frontendmentor.io) ajuda você a melhorar suas habilidades de programação construindo projetos realistas.

## O desafio

Seu desafio é construir este componente de QR code e deixá-lo o mais próximo possível do design.

Você pode usar quaisquer ferramentas que quiser para completar o desafio. Se quiser praticar algo específico, fique à vontade para usar.

### Quer apoio no desafio?

[Entre na nossa comunidade](https://www.frontendmentor.io/community) e faça perguntas no canal **#help**.

## Onde encontrar tudo

Sua tarefa é construir o projeto conforme os designs na pasta `/design`. Você encontrará versões para mobile e desktop do design.

Os designs estão em formato JPG estático. Usar JPG significa que você precisa usar seu melhor julgamento para estilos como `font-size`, `padding` e `margin`.

Se quiser o arquivo de design Figma para ganhar experiência com ferramentas profissionais e criar projetos mais precisos rapidamente, você pode [assinar como membro PRO](https://www.frontendmentor.io/pro).

Você encontrará todos os ativos necessários na pasta `/images`. Os arquivos já estão otimizados.

Também há um arquivo `style-guide.md` contendo informações necessárias, como paleta de cores e fontes.

## Usando assistentes de codificação com IA

Incluímos dois arquivos para ajudar se estiver usando assistentes de codificação com IA (como Claude, GitHub Copilot, Cursor etc.):

- `AGENTS.md` - Contém instruções detalhadas para assistentes de IA sobre como ajudá-lo neste desafio. É adaptado ao nível de dificuldade do desafio, para orientar de forma adequada (mais suporte para iniciantes, mais independência para avançados).
- `CLAUDE.md` - Arquivo indicativo que direciona ferramentas baseadas em Claude para as instruções em `AGENTS.md`.

**Como usar:** não precisa fazer nada! Muitos assistentes detectam esses arquivos automaticamente, leem e ajustam o comportamento para serem um parceiro de aprendizado melhor — guiando para soluções em vez de simplesmente dar as respostas.

**Nota:** Esses arquivos existem para ajudar você a *aprender*, não para fazer todo o trabalho. A IA é instruída a fazer perguntas, dar dicas e explicar conceitos em vez de escrever soluções completas.

## Construindo seu projeto

Use o fluxo de trabalho com que se sinta confortável. O processo sugerido abaixo é opcional:

1. Inicialize seu projeto como repositório público no [GitHub](https://github.com/). Assim fica mais fácil compartilhar com a comunidade se precisar de ajuda. Se não souber fazer isso, [leia este recurso Try Git](https://try.github.io/).
2. Configure o repositório para publicar em um endereço web. É útil para pedir ajuda, pois você pode compartilhar URL do site e do repositório. Há várias maneiras, e algumas recomendações estão abaixo.
3. Analise os designs para planejar o projeto. Isso ajuda a pensar em classes CSS reutilizáveis.
4. Antes de adicionar estilos, estruture o conteúdo com HTML. Escrever HTML primeiro ajuda a focar na estrutura correta.
5. Escreva estilos base, incluindo estilos gerais como `font-family` e `font-size`.
6. Comece pelas partes de cima da página e vá descendo. Só passe para a próxima seção quando estiver satisfeito com a anterior.
